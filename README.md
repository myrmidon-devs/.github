# 🛡️ Myrmidon Devs:

Bienvenido al centro de operaciones de **Myrmidon Devs**. Este es un espacio exclusivo para desarrolladores de confianza donde compartimos conocimiento, herramientas y proyectos experimentales. 

Como los antiguos mirmidones, aquí trabajamos con disciplina, lealtad y excelencia técnica.

---

## 📜 Reglas

Para mantener el orden en nuestra base de datos, seguimos estas directrices:

1. **Autonomía con Respeto:** Tienes total libertad en tus propios repositorios. Sin embargo, para colaborar en los de otros compañeros, es **obligatorio** el uso de Pull Requests.
2. **Cultura de Review:** Nadie fusiona código en la rama `main` sin que al menos otro miembro haya revisado los cambios. ¡Aprendemos todos de todos!
3. **Nomenclatura:** Intenta usar prefijos para que nuestros repositorios estén organizados:
    * `plugin-` para extensiones o librerías.
    * `skill-` para skills de Copilot.
    * `guide-` para guías y tutoriales.

## 🛠️ Cómo contribuir

1. **Crea una rama:** `git checkout -b feature/nueva-mejora`
2. **Haz tus commits:** Sé descriptivo con tus mensajes.
3. **Abre un Pull Request:** Etiqueta al dueño del repositorio para que revise tu aporte.

## 🏛️ Estructura de la Organización

* **Strategos:** Administración y arquitectura.
* **Myrmidons:** Desarrolladores y artesanos del código.
